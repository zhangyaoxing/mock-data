# MockData - Flexible Mock Data Generator

[![Python Version](https://img.shields.io/badge/python-3.11+-blue.svg)](https://www.python.org/downloads/)
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)

A flexible and powerful mock data generator based on JSON schemas generated by [MongoDB Compass](https://www.mongodb.com/docs/compass/). Built on top of [Faker](https://faker.readthedocs.io/), MockData generates realistic test data and output to one of the following:
- MongoDB
- Kafka
- file-based outputs in [EJSON](https://www.mongodb.com/docs/manual/reference/mongodb-extended-json/) format.

## Dependencies
- Python >= 3.11
- Python dependencies are in `pyproject.toml`
- cmake (Optional)

## Installation
```bash
# Clone the repository
git clone https://github.com/zhangyaoxing/mock-data.git
cd mock-data

# Install in development mode
pip install -e .

# Or install with dev dependencies
pip install -e ".[dev]"
```

## Quick Start
Schemas are defined using [JSON Schema](https://json-schema.org/) with MongoDB BSON type extensions. MockData utilizes the schemas exported from Compass. In Compass UI, you can export the schema as `JSON`, then let MockData use the exported JSON schemas to generate dummy data.

```
usage: mockdata [-h] [-s SCHEMAS] [-t OUTPUT_TYPE] [-n NUM_OBJECTS] [-r] [target]

Mock data generator based on JSON schemas.

positional arguments:
  target                Depending on the -t argument, specify folder for ejson, connection string (mongodb:// or mongodb+srv://) for
                        mongodb, or bootstrap server for kafka.

options:
  -h, --help            show this help message and exit.
  -s SCHEMAS, --schemas SCHEMAS
                        Path to the directory containing JSON schema files, or json schema file generated by Compass. (default: schemas)
  -t OUTPUT_TYPE, --output-type OUTPUT_TYPE
                        Output type (default: ejson). Options: ejson, mongodb, kafka
  -n NUM_OBJECTS, --num-objects NUM_OBJECTS
                        Number of objects to generate (default: 100)
  -r, --refresh         Refresh the ChromaDB collection with Faker methods. Use this if you have updated Faker or added custom providers.
                        Default: False
```
### Examples
```bash
# generate dummy data into the output/ folder as ejson
mockdata -s schemas/BookStore.json -n 50 -t ejson output/
# generate dummy data into MongoDB
mockdata -s schemas/BookStore.json -n 50 -t mongodb mongodb://localhost/
# generate dummy data into Kafka
mockdata -s schemas/BookStore.json -n 50 -t kafka localhost:9092
```

## Documentation
### 1. How It Works?
MockData uses the names of generator methods to generate embeddings and store them in [ChromaDB](https://www.trychroma.com/). When generating dummy data for a schema, we use the field name to do vector search against the methods collection to find the most meaningful generator methods. Dummy value is then generated by this method. Try using meaningful names like `email`, `address`, `phone` to let MockData make more accurate guesses.

### 3. Customize Generator Method
If the result doesn't turn out good enough, you can customize which Faker method to use by using the `description` field. The `description` field is designed to explain the field. We are utilizing this field to inject the method names by surrounding them with `#`.

**Format**

```
#<method_name>(<param1>, <param2>, ...)#
```

**Examples**

- `#name()#` - Generate a random name
- `#email()#` - Generate a random email
- `#random_int(18, 80)#` - Random integer between 18 and 80
- `#date_of_birth(None, 18, 25)#` - Random birthday, age 18-25
- `#sentence(50)#` - Random sentence with 50 words
- `#uuid4()#` - Generate UUID

See [Faker Standard Providers](https://faker.readthedocs.io/en/stable/providers.html) for all available methods.

#### Using Enums
Although this is currently not supported by Compass, you can uise standard JSON Schema `enum` for predefined values:

```json
{
  "status": {
    "bsonType": "string",
    "enum": ["pending", "active", "completed", "cancelled"]
  }
}
```
